Blockquotes

The blockquote element represents content that is quoted from another source, optionally with a citation which must be within a footer or cite element, and optionally with in-line changes such as annotations and abbreviations.

Blockquote without attribution

Tiam, ad mint andaepu dandae nostion secatur sequo quae. Note that you can use Markdown syntax within a blockquote.

Code block with backticks

/* RENDER PIPELINE
  There should exist render pipelines, these would be pipelines that 
  contain a shader program, a queue of renderables etc.
  
  Each pipeline will iterate over its queue each frame, rendering
  each renderable with its own shader program.

  There should be an easy way to register a new render pipeline by
  calling some method that takes in a shader file path and other
  meta-data and sets up a rendering pipeline that can be used at will.

  Two pipeline varients should exist, one that keeps its render queue
  alive after each iteration, rendering them over and over until
  manually removed from the queue.

  The other type of pipeline should automatically drop a renderable
  from its queue after each iteration.

  For example:
  // create a new pipeline
  init_render_pipeline("myshader.txt", CLEAR_QUEUE | SOME_FLAG)

  // add a renderable to the pipeline queue
  pipeline_add(PIPE_QUAD, my_renderable)
*/

static SDL_GLContext gl_context = NULL;

static void cleanup()
{
  PDBG("Cleaning up renderer_gl\n");

  if (gl_context != NULL) {
    SDL_GL_DeleteContext(gl_context);
    gl_context = NULL;
  }
}
